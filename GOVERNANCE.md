# Governance

## Maintainers

- **Warren Koch** — warrenkoch@gmail.com (project lead)

## Decision Process

- **Bug fixes & minor features**: Maintainer approval on PR
- **Breaking changes**: RFC-style proposal in GitHub Issues, minimum 7-day comment period
- **Architecture decisions**: Documented in project DECISIONS.md

## Roadmap

Managed via GitHub Issues and the project TODO. Priorities are driven by:
1. Security and correctness
2. FIPS/standards compliance
3. Developer experience
4. Feature expansion

## Status

This project is early-stage (v0.x). Governance will evolve as the community grows. The current model is benevolent dictator — simple and fast while the contributor base is small.
